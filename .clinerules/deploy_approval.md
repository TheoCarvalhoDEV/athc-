# Regra de Aprovação de Deploy e Commit

Antes de realizar qualquer deploy ou git commit:
1. Você DEVE informar ao usuário o resumo das alterações e quais arquivos foram modificados.
2. Você DEVE solicitar confirmação explícita antes de executar comandos de deploy ou commit.
3. Só prossiga após o usuário autorizar explicitamente o deploy ou o commit.
